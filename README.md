# rheii-landing

The marketing site at **https://www.rheii.com**. Edit anything in here, push to `main`, and it goes live in ~1 minute via Vercel.

---

## First-time setup (do once)

You need three things installed:

1. **Node.js** — download from https://nodejs.org (pick the LTS version, click through the installer)
2. **Git** — comes with Mac by default; type `git --version` in Terminal to check
3. **A code editor** — recommended: [Cursor](https://cursor.com) (built-in AI; you can describe changes and it writes the code)

Then in Terminal:

```bash
git clone https://github.com/WilsonLimSet/rheii-landing.git
cd rheii-landing
npm install
```

That last step takes ~30 seconds and installs all the libraries the site uses.

## Daily workflow

### See your changes locally

```bash
npm run dev
```

Open http://localhost:3000 in your browser. Edit a file, save, and the page reloads automatically. **Always preview locally before pushing to production.**

Stop the dev server with `Ctrl+C` when you're done.

### Common edits

- **Change copy / wording**: open the relevant `.tsx` file, find the text in quotes, edit it.
- **Change an image**: drop the new image into `public/images/landing/` (or relevant subfolder), then update the file name where it's referenced in the component.
- **Change colors / fonts**: see `src/app/globals.css` and `tailwind.config.ts`.

If you can't figure out where something lives, ask Cursor / Claude Code: "where is the text 'Growth Made Visible' in this codebase?" — the AI will find it.

## Publishing a change to www.rheii.com

The safe way (recommended for non-trivial changes):

```bash
git checkout -b your-change-name        # make a branch
# ... edit files ...
git add -A
git commit -m "describe your change"
git push -u origin your-change-name      # push the branch
```

Then on GitHub, click **"Compare & pull request"**. Vercel will automatically build a **preview URL** (something like `rheii-landing-abc123.vercel.app`) — share that with Wilson for a quick eyeball before merging.

When the PR is merged into `main`, the change goes live at https://www.rheii.com within ~60 seconds.

The faster way (only for small text fixes you're confident about):

```bash
git add -A
git commit -m "fix typo in hero text"
git push
```

If you push directly to `main` and the build fails, **www.rheii.com keeps serving the previous version** — Vercel only swaps to a new deploy after a successful build. So nothing visibly breaks; the failed build just sits in the Vercel dashboard.

## Vibe coding with AI

In Cursor or Claude Code, you can describe a change in plain English:

> "Make the headline on the hero section say 'Growth, made visible.' instead of 'Growth Made Visible'"

The AI will find the file, propose the edit, and you approve it. **Always run `npm run dev` and look at the change in your browser before pushing.** AI sometimes edits the wrong file or breaks something subtle, and the local preview catches it.

## When something breaks

| Problem | Fix |
|---|---|
| `npm run dev` says "port 3000 already in use" | Either close the other terminal running it, or run `npm run dev -- --port 3001` |
| `npm install` fails with weird errors | Delete `node_modules` and `package-lock.json`, then run `npm install` again |
| Page looks broken locally | Stop dev server (Ctrl+C), run `npm run dev` again |
| Pushed to `main` but site didn't update | Check https://vercel.com/wilsonlimsets-projects/rheii — there'll be a red "Failed" build with the error. Ping Wilson. |

For anything else: ping Wilson, or ask the AI in Cursor / Claude Code.

---

## What NOT to do

- **Don't edit files inside `node_modules/`** — those are libraries, regenerated by `npm install`.
- **Don't edit `package-lock.json` by hand** — let `npm install` manage it.
- **Don't commit `.env` files** if you ever create one — `.gitignore` blocks them, but double-check.
- **Don't delete files in `public/images/Font/`** — they're the custom fonts the site loads.

## Tech stack (FYI)

- **Next.js 14** — React-based framework
- **Tailwind CSS** — for styling
- **Framer Motion** — for animations
- **Hosted on Vercel** — auto-deploys from this `main` branch

You don't need to understand any of these to make changes — just edit the file, save, and check the browser.
